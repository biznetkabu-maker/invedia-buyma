# invedia-automation 運用マニュアル

> **対象読者**: BUYMA バイヤーとして実際にこのシステムを使う人向け。  
> コードの知識は不要。ターミナル（コマンド操作）の基本操作があれば十分。

---

## システムが何をするか（全体像）

```
【あなたがやること】             【このシステムがやること】
────────────────────             ──────────────────────────────────────────
1. 商品を見つけて登録              ← intake.py が検索URL生成・価格比較・判定を補助
2. BUYMAに手動で出品               ← 出品フォームへの入力は人間が行う
3. 在庫切れ商品を停止              ← main.py が「停止中」に変えて LINE で通知

↑ここだけやれば動く              ↓ここは全自動（GitHub Actions）

                                  毎時: 価格・在庫を全サイトから自動スクレイプ
                                        利益再計算
                                        シートのステータスを自動更新
                                        お宝商品（利益 ≥ ¥30,000）を LINE 通知
                                        為替レートを自動取得（AUTO_FOREX=true）
```

## 初回セットアップ（一度だけやる）

### 1. Google Sheets の準備

1. [Google Sheets](https://sheets.google.com) で新規スプレッドシートを作成する
2. URLからスプレッドシートIDをコピーする
   ```
   https://docs.google.com/spreadsheets/d/【ここがID】/edit
   ```
3. シート名（タブ）を `02_Purchase_Control` にする

### 2. Google サービスアカウントの作成

1. [Google Cloud Console](https://console.cloud.google.com/) を開く
2. 新規プロジェクトを作成 → 「Google Sheets API」を有効化
3. 「認証情報」→「サービスアカウント」を作成
4. サービスアカウントのメールアドレス（`xxx@xxx.iam.gserviceaccount.com`）をコピー
5. スプレッドシートの「共有」→ そのメールアドレスに「編集者」権限を付与
6. サービスアカウントの「鍵」→ JSON 形式でダウンロード → `credentials.json` という名前で保存

### 3. 環境変数ファイルの作成

```bash
cp .env.example .env
```

`.env` を開いて最低限の設定を記入：

```bash
SPREADSHEET_ID=xxxxxxxxxxxxxxxxxx   # Step 1 でコピーしたID
CREDENTIALS_PATH=credentials.json   # ローカル実行時
WORKSHEET_NAME=02_Purchase_Control
```

### 4. ライブラリのインストール

```bash
pip install -r requirements.txt
playwright install chromium
```

### 5. GitHub Actions の設定（自動実行用）

GitHub リポジトリの「Settings」→「Secrets and variables」→「Actions」に以下を登録：

| 名前 | 内容 |
|------|------|
| `GOOGLE_CREDENTIALS_JSON` | `credentials.json` の内容をそのままコピペ |
| `SPREADSHEET_ID` | スプレッドシートID |
| `LINE_NOTIFY_TOKEN` | LINE Notify のトークン（任意・通知が欲しい場合） |

## 日常の使い方

### ケース1: 新商品を登録する（週1〜2回）

**流れ: 候補商品を決めた → `intake.py` で仕入先・利益を確認 → シートへ追加**

詳しい取り扱い（Step 1 の入力項目の意味、為替、Step 2〜6 の挙動）は **`INTAKE_MANUAL.md`** を参照してください。

```bash
python3 intake.py
```

#### 実際の画面の流れ（6ステップ概要）

```
============================================================
  BUYMA 商品取り込みツール
============================================================

── Step 1: 基本情報の入力（手動） ───────────────────────────
  ブランド名・商品名・カテゴリ・モデル年・BUYMA予定販売価格（円）・通貨
  型番（任意）・BUYMA商品URL（型番の自動取得用、任意）
  → 為替は API 自動取得（失敗時は手入力）
  入力例・各項目の意味は **`INTAKE_MANUAL.md`** を参照

── Step 2: BUYMA 需要確認（自動） ─────────────────────────
  Playwright で BUYMA 公開検索 → お気に入り・競合・価格帯などを表示

── Step 3: 仕入先URL の確認（半自動） ─────────────────────
  15サイト分の検索URLを表示 → ブラウザで商品ページを開き URL を貼る（カンマ区切り可）

── Step 4: N件のURLを並列スクレイプ（自動） ────────────────
  最安・在庫あり優先で仕入れ先候補を選定（または URL なしで現地価格を手入力）

── Step 5: 仕入れ判断（自動） ───────────────────────────────
  A〜E グレード・利益率・改善提案を表示

── Step 6: スプレッドシートへ追加 ───────────────────────────
  D/E のときだけ「それでも追加？」と確認 → シートに追記
```

**型番を入力した場合:** Step 4 の一覧で ✅一致 / ❌不一致 が表示されます。不一致のURLは最安選定から除外されます。

**追加後にやること:**

- グレードと `INTAKE_MANUAL.md` のトラブルシューティングを踏まえ、必要ならシート上の価格・URL を手で微修正する
- A または B など問題なければ → **BUYMAに手動で出品**（規約遵守のため、当面は出品フォームを人手で操作することを推奨）

### ケース1b: BUYMA 候補をブックマークレットで溜める（任意）

BUYMA の検索・ランキングページで候補 URL を素早く集めたいとき:

1. `bookmarklets/install_bookmarklet.html` をブラウザで開き、ブックマークを登録（**Python不要**）
2. BUYMA 一覧ページでブックマークを実行 → **「TSV をコピー」**
3. リポジトリルートで（**メモ帳への保存は不要**）:

```bash
python3 scripts/import_buyma_tsv.py
```

Windows なら `scripts\import_clipboard.bat` をダブルクリックでも同じです。

→ シートに **在庫ステータス = `BUYMA候補`** の行が追加される（`仕入れURL` は BUYMA 商品ページ・参照用）。  
`main.py` は `buyma.com` の URL をスクレイプしない。本登録は **`python3 intake.py`** で仕入先 URL・価格を埋める。

詳細: `bookmarklets/README.md` / `NEW_CHAT_HANDOFF.md`

---

### ケース1c: 仕入れ検討を自動化する（BUYMA候補 → 仕入先URL）

**手動で型番・海外URLを貼らなくてよいモード**（Playwright 使用・1件あたり 1〜3 分）。

| やりたいこと | コマンド / bat |
|--------------|----------------|
| シートの `BUYMA候補` 行を一括処理 | `3_候補_自動仕入れ検討.bat` または `py intake.py --auto-sheet` |
| 1件だけ試す | `py intake.py --auto-sheet --limit 1` |
| BUYMA 商品 URL 1件だけ | `py intake.py --auto-buyma https://www.buyma.com/items/...` |

**自動で行うこと:** BUYMA ページからブランド・商品名・型番取得 → 需要確認 → 主要5サイト（SSENSE / MYTHERESA / FARFETCH / NET-A-PORTER / 24S）の検索結果から商品URL収集 → 並列スクレイプ・型番照合 → グレード判定 → シート更新（`在庫ステータス` は `出品前`）。

**対象行:** `在庫ステータス = BUYMA候補` かつ `仕入れURL` が `buyma.com` の商品ページ URL。

**フォールバック:** 仕入先 URL が取れない・グレード D/E のときはシートを更新せず、従来どおり `py intake.py`（対話）で URL を貼る。

D/E もシートに反映したい場合: `--include-low-grades`

### ケース2: BUYMAに出品する（手動）

**システムは出品しない。人が BUYMA の画面から行う。**

> ⚠️ `buyma_automator.py` の自動出品機能（`AUTO_LIST_GRADE`）は BUYMA の利用規約との整合性を確認してから使用すること。**当面は手動出品を推奨。**

出品時に使う情報はシートから確認できる：
- 商品名、ブランド、現地価格、利益計算結果
- `checklist_template.md` を印刷して確認項目を一つずつチェックする

### ケース3: 在庫切れ商品を停止する（通知が来たら）

LINE に以下の通知が来たとき：

```
⛔ 停止中になりました: CELINE トリオバッグ スモール ブラック
仕入れURL: https://www.ssense.com/...
```

**やること:** BUYMA の「マイページ」→「出品中の商品」→ 該当商品を「出品停止」にする

> システムはシートのステータスを「停止中」に変えるだけ。BUYMA 側の停止は手動。

### ケース4: お宝商品の通知が来た

利益が `LINE_PROFIT_THRESHOLD`（デフォルト ¥30,000）を超えたとき LINE に通知が届く。

```
🏆 お宝商品を発見しました！
商品: HERMÈS バーキン 30 エトゥープ
利益: ¥180,000 (22.5%)
URL: https://www.harrods.com/...
```

**やること:**
1. URLを開いて在庫を確認
2. `python3 evaluate.py` で詳細評価（物流・需要・リスクを手動入力）
3. A/B グレードなら仕入れ検討 → **BUYMAに手動出品**

---

### ケース5: 要確認の商品に対応する

シートのステータスが「要確認 (利益率 8.3%)」になっている場合：

**原因:** 仕入先の価格が上がり、現在の BUYMA 販売価格では目標利益率を下回っている

**対応の選択肢:**
1. BUYMA の出品価格を上げる → シートの `BUYMA販売価格` 列も更新
2. より安い仕入先を探す → `intake.py` で再度候補URLを入力して最安値を更新
3. そのまま様子を見る（ステータスを手動で「出品中」に戻すことも可能）

---

### ケース6: 型番不一致の通知が来た

```
⚠️ 型番不一致を検出しました (1件)
  - CELINE トリオバッグ: シート型番=ARC58-BLK / 仕入先=OTHER (URL: https://www.ssense.com/...)
```

**原因:** シートの `型番` 列と、仕入先ページから取得した商品ID（JSON-LD の sku 等）が一致しない。

**やること:**
1. 仕入れURLが別商品・別カラーになっていないかブラウザで確認
2. 正しい商品ページURLに `仕入れURL` を更新（`intake.py` で再登録も可）
3. BUYMA側の型番が誤っていれば `型番` 列を修正

シートのステータスは自動で **要確認 (型番不一致)** になります。

---

### ケース7: スクレイパー異常の通知が来た

```
⚠️ スクレイパー異常を検出しました（2件）
  - CELINE バッグ: 3回連続でステータス不明 (URL: https://ssense.com/...)
  - Balenciaga スニーカー: 3回連続でステータス不明
```

**やること:**
1. 対象URLをブラウザで手動確認（サイトが開けるか・価格が表示されているか）
2. URLが変わっていたら → シートの `仕入れURL` 列を新しいURLに更新
3. サイトが動いているなら → しばらく待って様子を見る（一時的なブロックの可能性）

## シートの見方と操作

### ステータス別の対応一覧

| ステータス | 意味 | あなたがやること |
|-----------|------|----------------|
| `出品前` | intake.py で追加したがBUYMAにはまだ出品していない | 需要確認 → BUYMA に出品 |
| `BUYMA候補` | ブックマークレット等で候補のみ登録 | `intake.py` で本登録（仕入れURL・価格） |
| `出品中` | 在庫あり・利益 OK | 何もしない |
| `停止中` | 仕入先で在庫切れ | BUYMA の出品を停止する |
| `要確認 (利益率 X%)` | 利益が目標以下 | 価格見直し or 仕入先変更 |
| `要確認 (型番不一致)` | シート型番と仕入先 ID が不一致 | URL・型番を確認（ケース6） |
| （空欄） | URL なし or スクレイプ失敗 | URLを確認・更新 |

### 手動で変更できる列

| 列 | 手動変更してよい場合 |
|----|-------------------|
| `仕入れURL` | 仕入先サイトが商品ページを移動したとき |
| `BUYMA販売価格` | 競合確認後に価格を調整したとき |
| `為替` | AUTO_FOREX=false のときのみ（true なら自動） |
| `在庫ステータス` | 緊急で「出品中」に戻したいときなど |
| `候補URLs` | 新しい仕入先URLを追加したいとき（カンマ区切りで追記） |
| `型番` | intake で登録した Style ID / SKU（main.py が照合に使用） |

## トラブルシューティング

### 「シートに接続できない」

```
認証情報ファイルが見つかりません: credentials.json
```
→ `credentials.json` をこのフォルダに置いているか確認。パスは `.env` の `CREDENTIALS_PATH` で変更可能。

### 「スクレイプできない・価格が取れない」

→ 対象URLをブラウザで直接開いて確認。

- **ページが開けない** → IPがブロックされている可能性。時間をおくか、プロキシ設定（`PROXY_LIST`）を検討。
- **ページは開けるが価格が出ない** → サイトのHTML構造が変わった可能性。`scraper/strategies/` 内の該当ファイルのセレクタを更新する必要がある。

### 「為替が取得できない」

```
為替API取得失敗。古いキャッシュを使用
```
→ インターネット接続を確認。一時的なAPIの障害なら30分〜1時間後に自動回復。

### 「GitHubActionsが動いていない」

→ GitHub の「Actions」タブで最新の実行ログを確認。「Secrets」に `GOOGLE_CREDENTIALS_JSON` と `SPREADSHEET_ID` が登録されているか確認。

## 重要な注意事項まとめ

| ⚠️ 注意事項 | 詳細 |
|------------|------|
| **BUYMAへの出品は手動で** | `AUTO_LIST_GRADE` による自動出品は利用規約を個別確認してから使用すること |
| **`credentials.json` は絶対に Git に入れない** | `.gitignore` に含まれているが、絶対に確認すること |
| **「停止中」を見たらすぐBUYMAで停止** | システムはシートを更新するだけ。BUYMA 側の操作は人間が必ず行う |
| **画像は自分で用意する** | 仕入先サイトの画像をそのまま使用しないこと（著作権） |
| **`OPERATION_MODE=discovery` は利用規約確認後に** | デフォルトは `monitor`。`discovery` はBUYMAページのスクレイプを含む |
